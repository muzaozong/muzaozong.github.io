Linux ssh执行远程命令无法获取环境变量
发表于 2018-08-16| 更新于 2019-12-13|linux
字数总计: 261|阅读时长: 1 分钟| 阅读量:

原因

通过ssh执行远程命令会使用Bash的non-interactive + non-login shell模式，它会创建一个shell，执行完脚本之后便退出，不再需要与用户交互。它不会去执行/etc/profile文件，而会去用户的HOME目录检查.bashrc并加载。

解决方案

  1. 在.bashrc中添加export，但是必须添加在如下语句上方
  2. 在命令中加入export语句，例如: ssh user@host 'export JAVA_HOME=xxxx && ls'
  3. 在命令中加入source语句，例如: ssh user@host 'source xxxx && ls'

进一步

如果我们还需要在远程命令中执行sudo操作，那么环境变量又会失效

解决方法
