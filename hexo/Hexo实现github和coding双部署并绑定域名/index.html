Hexo实现github和coding双部署并绑定域名
发表于 2017-10-23| 更新于 2019-12-13|Hexo
字数总计: 325|阅读时长: 1 分钟| 阅读量:

创建项目

github

在github上创建一个名称为username.github.io(例如我的就是comeacrossyun.github.io)的项目，项目的setting中打开Github Pages选项，启用master分支作为部署分支。

coding

与github上类似，在coding上创建一个名为username(例如我的就是cayun)的项目，点开项目->代码->静态Pages->启用master作为部署分支->绑定自定义域名。

设置本地_config.yml

在站点根目录下的_config.yml文件中填写deploy配置。

需要注意的是，github的好像只能使用git地址，不能使用https地址。因为我们同时使用两个git仓库的原因，所以每个type之前的”-“千万不能丢。

本地添加CNAME文件

在source目录下新建一个名为CNAME的文件，里面写上自己的域名。

设置域名解析

在域名服务商处设置域名解析，分别添加两条CNAME记录：

  • CNAME www 默认 cayun.coding.me
  • CNAME www 海外 comeacrossyun.github.io

上面是我的记录，请根据您自己的情况自行填写。

注意事项

本文并未涉及hexo的具体部署方式等。如需请自行网上搜索。
