    前言

    在学习Java的过程中，主要有接触到三种锁，于是很好奇这三种锁的性能，本次就是对这三种做一次定量的性能测试。

    基本信息

    Java版本：JDK1.8

    并发数：300

    读操作时间: 0.02ms

    写操作时间：读操作时间的1倍/2倍

    测试代码：见附录

    三种锁

    • ReentrantLock
    • synchronized
    • ReentrantReadWriteLock

    测试结果

    测试结果仅供参考，时间单位ms

    写操作与读操作耗时相等

    读写比例 ReentrantLock synchronized ReentrantReadWriteLock
    全读 7550 7629 170
    99:1 7635 7890 1659
    49:1 7715 7719 3128
    9:1 7724 7702 5552
    4:1 7532 7573 7177
    2:1 7761 7821 7958
    1:1 7613 7750 8212
    全写 7656 7664 7622

    写操作耗时为读操作耗时2倍

    读写比例 ReentrantLock synchronized ReentrantReadWriteLock
    全读 7515 7668 116
    99:1 7768 7883 1849
    49:1 7768 7795 3285
    9:1 8317 8312 6126
    4:1 9070 9120 9060
    2:1 9856 10057 10608
    1:1 10979 11266 12328
    全写 14551 14735 14545

    结论

    1. Lock与synchronized的耗时基本一致
    2. 读写锁的性能在读操作远大于写多操作的时候性能会比其他两种方式更优
