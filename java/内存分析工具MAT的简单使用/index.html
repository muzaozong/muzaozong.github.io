JVM堆dump

使用参数-XX:+HeapDumpOnOutOfMemoryError

添加JVM参数-XX:+HeapDumpOnOutOfMemoryError。该参数会使得程序在发生OOM之后产生一个堆dump文件，默认是生成在应用程序的启动目录(就是执行java ……命令的那个目录)。可以通过-XX:HeapDumpPath={dir}来指定位置。

MAT

MAT 全称 Eclipse Memory Analyzer

下载地址 https://www.eclipse.org/mat/downloads.php

使用

1. 打开Heap Dump

这个对话框可以直接忽略，点取消。

2. Leak Suspects

Leak Suspects是用来分析内存泄漏原因的。

入口

Overview

首先是一块概览图，这是一块饼状图，深色区域表示有可能出现内存泄露的怀疑对象的大小。

描述

此处是suspect的简单描述，点Details可以看到细节。

细节

描述部分点Details可以看到具体的细节

细节部分主要包含如下几个部分

Shortest Paths To the Accumulation Point

该部分显示的是从一个GC root到累积对象的最短路径

Accumulated Objects in Dominator Tree

这部分主要是内存中的累积对象的内容。

Accumulated Objects by Class in Dominator Tree

(这部分与下面部分暂未搞懂区别)

在这个部分以及下面的部分中，我们可以看到是哪些对象堆积在内存中。

All Accumulated Objects by Class

System Overview

主要包含如下部分

  • 堆dump的信息
  • 系统属性
  • 堆dump时线程的状态概览
  • 顶级消费者——例如：最大的对象、类、类加载器、包等
  • 类直方图

3. Histogram

列出内存中的对象，个数及大小

Shallow Heap & Retained Heap

shallow heap

指对象自身所占用的内存大小，不包含引用的对象所占的内存大小。

retained heap

指如果当前对象被GC后，可以释放的内存大小。

用一张来自MAT官方帮助文档上的图举个例子，当E被回收后，那么G也可以被回收，但是H却不可以，因为H还有来自其他对象的引用，那么Retained Heap就指的是E+G的大小。

4. Dominator Tree

支配树与直方图的角度不同，直方图是基于类的角度，支配树是基于对象的角度，支配树可以更方便地看出其引用关系。

5. Top Consumer

通过图形列出最大的对象、类加载器等。
